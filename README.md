    <div class="section-2"  id="47186029_VisualStudio2010%2C2012and2013IDEPlugins-VisualStudioPlugin"  >
        <h2>Visual Studio Plugin</h2>
    <div class="section-3"  id="47186029_VisualStudio2010%2C2012and2013IDEPlugins-Installation"  >
        <h3>Installation</h3>
    <p>
All you need to do is to    </p>
<ol class=" "><li class=" ">    <p>
extract the files of the <a href="Visual_Studio_2010,_2012_and_2013_IDE_Plugins.html#47186029_VisualStudio2010%2C2012and2013IDEPlugins-plugindetails">dynaTrace Plugin depending on your dynaTrace version</a> to your <i class=" ">My Documents\Visual Studio 2010\Addins</i> directory. If the <i class=" ">Addins</i> directory does not exist, create it.    </p>
</li><li class=" ">    <p>
Make sure that the downloaded files do not have the &quot;blocked&quot; file attribute under Windows 7. Change that to unblocked    </p>
</li><li class=" ">    <p>
If running Windows XP, after unpacking the files, it might be necessary to change the file protection to RW.  If VS starts with an error related to this plugin, then the protection isn't set generously enough, or the 'blocked' flag is likely set if on Windows 7.    </p>
</li><li class=" ">    <p>
<strong class=" ">Now start Visual Studio as Administrator</strong> - this is required as the dynaTrace plugin requires access to the registry.  (if on Windows XP, simply set all the plugin files to be RW for all users)    </p>
</li><li class=" ">    <p>
The dynaTrace Visual Studio 2010 Plugin can be enabled through <i class=" ">Tools</i>-&gt; <i class=" ">Add-In Manager</i> in Visual Studio.    </p>
</li><li class=" ">    <p>
You need to have installed the dynaTrace .NET agent for launcher to work!!    </p>
</li></ol>    <p>
Note: If VS displays an error when starting and disables the plugin, go back to the steps above and verify the file protection issues listed above.    </p>
    <p>
Once the Add-In is loaded you will see two new menu entries in the Tools menu. One is to configure the dynaTrace Plugin (<i class=" ">dynaTrace Configuration</i>), the other one is to Launch the currently opened project by automatically injecting the dynaTrace .NET Agent into the launched application (<i class=" ">dynaTrace Launcher</i>).    </p>
    </div>
    <div class="section-3"  id="47186029_VisualStudio2010%2C2012and2013IDEPlugins-Whatthepluginprovides"  >
        <h3>What the plugin provides</h3>
    <p>
This plugin allows you to <strong class=" ">Lookup Source Code</strong> for a method traced by dynaTrace. From the dynaTrace Client you can select a method in the Methods or PurePath Dashlet and Lookup the method in Visual Studio. <strong class=" ">Please be aware that the code lookup only works for C# Source Code.</strong>    </p>
    <p>
The plugin also allows you to <strong class=" ">launch an application</strong> from the IDE and automatically inject the dynaTrace Agent to trace the launched application.    </p>
    <p>
The plugin also extends the Visual Studio Load Testing Feature with a new tab in the Web Test Result Viewer. When you test a web application and the requests made by Visual Studio return PurePath IDs (by using the Web Test Plugin or by specifying &quot;Always send dynaTrace HTTP Headers&quot; in the ASP.NET Sensor) then you can directly lookup a PurePath from the Result Viewer in Visual Studio    </p>
    <p>
Additional Online Documentation: <a href="Visual_Studio_2010,_2012_and_2013_IDE_Plugins.html">CodeLink AddIn for Visual Studio</a>    </p>
    </div>

        <h2>Web and Load Test Plugin</h2>
    <div class="section-3"  id="47186029_VisualStudio2010%2C2012and2013IDEPlugins-Installation.1"  >
        <h3>Installation</h3>
    <p>
Preconditions    </p>
<ol class=" "><li class=" ">    <p>
dynaTrace Client installed on local machine    </p>
<ul class=" "><li class=" ">    <p>
The plugin drives the session recording via REST calls to the client. The plugin tries to start the client when it is not running    </p>
</li></ul></li><li class=" ">    <p>
Configuration on the dynaTrace Client (connected server / profiles) already done    </p>
<ul class=" "><li class=" ">    <p>
the settings for the plugin will require a server- and profile name to record a session    </p>
</li></ul></li></ol>    <p>
Here are the steps to install this plugin:    </p>
<ol class=" "><li class=" ">    <p>
Download and copy / unzip the <a href="Visual_Studio_2010,_2012_and_2013_IDE_Plugins.html#47186029_VisualStudio2010%2C2012and2013IDEPlugins-plugindetails"> <i class=" ">dynaTrace Web Test Plugin for Visual Studio 2010 Load Testing</i> depending on your dynaTrace version</a> into your Load Testing Project.    </p>
</li><li class=" ">    <p>
Make sure that the downloaded files do not have the &quot;blocked&quot; file attribute under Windows 7. Change that to unblocked    </p>
</li><li class=" ">    <p>
In your project add a reference to copied / unzipped DLL(s).    </p>
</li></ol>    <p>
<strong class=" ">In a Web Test Definition</strong>    </p>
<ul class=" "><li class=" ">    <p>
Click on the &quot;Add Web Test Plugin&quot; toolbar button select the dynaTrace Web Test Plugin.    </p>
</li></ul>    <p>
<strong class=" ">In a Load Test Definition</strong>    </p>
<ul class=" "><li class=" ">    <p>
Add a Load Test Plugin that will enable Start/Stop Session Recording    </p>
</li><li class=" ">    <p>
<strong class=" ">Attention:</strong> the properties &quot;Server&quot; and &quot;SystemProfile&quot; are <i class=" ">case-sensitive</i>! Be sure to use the exact string like shown in the dT-Client.    </p>
</li></ul>    </div>
    <div class="section-3"  id="47186029_VisualStudio2010%2C2012and2013IDEPlugins-Whatthepluginprovides.1"  >
        <h3>What the plugin provides</h3>
    <p>
The Web Test Plugin will automatically tag Web Requests with the dynaTrace HTTP Header.<br/>The Load Test Plugin will automatically Start/Stop Session Recording during a load test.    </p>
